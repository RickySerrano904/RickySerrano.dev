# Designing for calm

Calm does not mean boring. It means removing the extra layers so the important
moments can breathe. This is a placeholder post that captures the kind of notes
I like to write after a build.

## TL;DR

- Start with constraints before adding personality.
- Reduce choices to make the UI feel obvious.
- Keep one signature detail so the product still feels alive.

## The problem

Most products accumulate patterns fast. New screens add new components, and the
UI gets noisy. Calm is the opposite of noise: it is a system that lets you see
what matters first.

## The approach

I audit the interface for friction, remove repeated decisions, then reintroduce
one intentional flourish. That might be a single accent color, a warmer typeface,
or a signature interaction.

## A tiny checklist

- One primary action per screen.
- One headline, one supporting sentence.
- One distinct component that signals the brand.

> Calm design respects attention. It is not minimal for its own sake.
